Network Compression Detection Tool

Languages: C & C++
Technologies: UDP/TCP, Entropy-based Analysis
View on GitHub

This project focused on identifying the presence of compression within a network path, with the goal of detecting and locating compression links. The project was implemented in two distinct applications:

  • Client/Server Application: Designed to run in cooperative environments where the server and client are both controlled. The server collects timing data of UDP packets with different entropy patterns to determine compression.
  • Standalone Application: Built for uncooperative environments. This tool can detect network compression without requiring any software to be installed or controlled on the remote host.
  • Packet trains with low and high entropy were analyzed to observe timing differences and infer compression activity.

This project required deep understanding of TCP/UDP, entropy, timing analysis, and real-world network constraints.
