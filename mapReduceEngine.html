Distributed Computation Engine (MapReduce)

Languages: Go, Protocol Buffers, JSON
Technologies: MapReduce, Distributed Systems, TCP Sockets, External Sorting
View on GitHub

This project is a distributed computation engine built for my CS 677 Big Data course by extending my distributed file system from Project 1 into a full MapReduce framework. Based on the course requirements, the system supports datatype-aware chunk partitioning, job submission, mapper and reducer execution, shuffle coordination, progress tracking, and storing final output back into DFS.

A dedicated Computation Manager receives jobs, finds the input file metadata from the controller, dispatches reducers and map tasks to storage nodes, and tracks job states through completion. The storage nodes execute mapper and reducer binaries, perform external sorting for intermediate data, stream shuffle partitions to reducers, and write reducer results back to the distributed file system.

Core features include:

  • Datatype-aware chunking for text files, preserving line boundaries for mapper input
  • Distributed Map, Shuffle, and Reduce execution with data-local task placement
  • External sorting and reducer partition merging for scalable intermediate processing
  • Job submission, progress monitoring, and final output persistence back into DFS

This project strengthened my understanding of distributed computation, scheduling around data locality, external sorting, and coordinating multi-stage data processing across multiple machines.
